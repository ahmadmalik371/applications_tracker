# AI Scoring & Ranking Algorithm

## Overview
The AI Score is a normalized value between `0` and `100` that represents how well a candidate matches a job description.

It is NOT calculated by asking an LLM to "score this resume out of 100". Instead, it is a composite score calculated via a deterministic Feature Engineering step followed by an ML Ranker (e.g., XGBoost).

## Feature Weights (Conceptual)

The ML model takes the following features as input:

1. **Semantic Similarity (40%)**: Cosine similarity between the Job Embedding and the Candidate Embedding.
2. **Required Skills Overlap (30%)**: Percentage of explicit required skills found in the candidate's profile.
3. **Experience Delta (20%)**: Difference in years between candidate's total experience and job requirement. (Penalties for being severely under-experienced).
4. **Education Match (10%)**: Does the candidate meet the degree requirements?

## Score Normalization
The output of the ML model (a probability score) is transformed into a user-friendly 0-100 score.

- `90-100`: Exceptional Match
- `75-89`: Strong Match
- `50-74`: Potential Match (Review needed)
- `< 50`: Unlikely Match

## Explainability
The UI must NEVER show just a number. It must include an explainability object generated by the ML Ranker (using SHAP values) and formatted by a lightweight LLM prompt:

```json
{
  "score": 85,
  "confidence": "high",
  "strengths": [
    "Matches 5/5 required technical skills",
    "High semantic similarity to job description"
  ],
  "weaknesses": [
    "Only has 3 years of experience (5 requested)"
  ]
}
```
