Authentication flow
- Backend: JWT access tokens (`create_access_token`) and long-lived refresh tokens stored server-side (`RefreshToken` records). Access tokens hashed with HS256 and settings.SECRET_KEY.
- Frontend: stores `access_token` and `refresh_token` in `localStorage` (see `frontend/src/lib/api-client.ts` and `auth-context.tsx`) and uses an axios interceptor to attach `Authorization: Bearer <token>`.
- Refresh: Axios intercept tries `POST /auth/refresh` when a 401 occurs, and updates localStorage with new tokens.
- Route protection: `AuthGuard` component on frontend redirects to `/login` when `isAuthenticated` is false; `AuthProvider` calls `/auth/me` to fetch current user.

Security note: tokens in `localStorage` are susceptible to XSS — consider using httpOnly cookies with CSRF protections for production.

Initial bugs / issues observed during scan (high priority)
1. Database authentication mismatch during local tests: `asyncpg.exceptions.InvalidPasswordError` — local `.env` and root `.env` show inconsistent passwords (`13851098` vs `ats_password`). This caused login 500 errors when backend attempted to connect to DB.
2. Missing Python dependency observed earlier: `argon2-cffi` missing in the environment (now present in `requirements.txt` and installed in `.venv` during audit).
3. Frontend JSX mismatches were present in job-related pages (`jobs/page.tsx`, `jobs/[jobId]/apply/page.tsx`) causing build errors — these were fixed during debugging by closing `AuthGuard` wrappers.
4. Some frontend routes use `apiClient.post('/candidates/jobs')`. Ensure backend `candidates.create_job` accepts the payload and that frontend users have the required roles. There are reports that job creation sometimes surface as "Failed to create job" in the UI while the job appears in lists — this indicates inconsistent response handling or a swallowed error (server returns non-2xx or error during embedding task triggering).
5. Notification counts and unread lists may be empty if migrations were not applied or if the notification task queue is not running. (There is an Alembic migration adding notifications tables.)
6. No `prisma` layer present despite original product note referencing Prisma.

Missing or risky areas
